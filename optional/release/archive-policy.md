# Archive Policy

Adapted from Wittgenstein's archive policy.

The repository should not keep stale surfaces just because they once existed. Choose one of three actions:

1. **Refresh** — update the file because it is still a live surface.
2. **Archive** — move it under an archive/history location because it is a useful receipt.
3. **Delete** — remove it because it is misleading or has no durable value.

Archive when a file explains why a decision happened, records a release, preserves a migration, or helps future maintainers understand risk.

Delete when a file duplicates current truth, contradicts current truth without historical value, or creates maintenance burden.

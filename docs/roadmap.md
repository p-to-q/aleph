# Roadmap

The roadmap is staged to keep the Hackathon build useful while leaving room for deeper search and observation work.

## v0: Hackathon Console

Goal: make the concept legible and usable without depending on a fragile model runtime.

Must ship:

- fixture-backed `AlephRun` data;
- target output input surface;
- compression slider over discrete candidate points;
- Pareto frontier summary;
- current prompt and model output panels;
- dashboard metrics: tokens, fit, stability, compression, leakage, frontier rank;
- token loss, search dial, waveform, attribution, exposure vectors, eval suite;
- clear fixture/simulated labels;
- README, core concept, architecture, research process, and verification docs.

Do not ship as v0:

- global shortest-prompt claims;
- real white-box claims without logits;
- full ARCA/GCG integration;
- persistence, accounts, or production deployment assumptions.

## v1: Real Black-box Runs

Goal: replace fixture candidate data with real repeated model calls.

Likely work:

- keep the active frontend path on `web/` and leave `apps/web/` archived;
- FastAPI `/runs` skeleton becomes real;
- hosted/local black-box adapter;
- candidate generation strategies;
- similarity metric implementation;
- repeated sampling for stability;
- leakage scoring;
- JSON run persistence;
- import/export saved runs.

## v2: White-box Scoring

Goal: make probability panels real for models that expose logits.

Likely work:

- teacher-forced target likelihood;
- token-level NLL;
- deletion ablation;
- prompt-token attribution;
- loss curve generated by real search steps;
- non-leaking mode enforcement.

## v3: Search Adapters

Goal: integrate stronger prompt search methods without making them the whole product.

Candidates:

- ARCA-style length scanning;
- GCG / nanoGCG-style hard prompt optimization;
- evolutionary search;
- semantic compression generators;
- draft-model filtering / probe-sampling-inspired acceleration.

## v4: Research Product

Goal: become a durable workbench for reverse prompt compression.

Possible surfaces:

- run gallery;
- cross-model transfer;
- model-relative complexity reports;
- exported research cards;
- dataset/batch mode;
- white-box probability microscope;
- stronger non-leaking evaluation.

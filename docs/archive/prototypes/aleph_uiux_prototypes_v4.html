统一测试文案
"Prompt is not an instruction. Prompt is a coordinate in the model’s library of possible continuations."
这次交付的目标：
不是再做“20 个方向草图”，而是把 Aleph 收束成一套更可执行的网页原型系统：

① 先做信息架构归类；
② 再把必须要有的二级可视化面板抽象成组件；
③ 最后给出 4 个更成熟、可观测、可阅读、且不抢主滑条与 Prompt 输入输出的网页版本。
Aleph · refined delivery

从“灵感拼贴”进入“可执行的产品原型”

我把你的意图重新归纳为：Aleph 的主舞台永远是滑条与 Prompt 输入/输出；其他可视化必须降级为可观测、可阅读、可交互的二级面板。 这些面板不是为了炫技，而是为压缩路径提供证据：它解释 为什么这个 prompt 在这里、它稳定吗、信息是否泄露、loss 和 attribution 如何变化。

参考风格：Aquin / Akyne 式研究型产品界面 主隐喻：Compression Slider / Pareto Frontier 定位：Launching Page + Product Page + Experiment Page 合一 方法：高复现风格 + 自主结构设计
Mini Dial / Loading
用小仪表盘替代传统 spinner，既符合研究仪器感，也不抢主界面。
必须要有 · Must-have
Core 主滑条 / Pareto 路径，且必须有明确的左右极：Shortest Found ↔ Explicit Reconstruction。
Core Target 输入、Current Prompt、Model Output / Compare 的三者闭环。
Core Dashboard 最少 5 项：Prompt Length、Fit、Stability、Compression、Leakage。
Core 二级可视化面板：Pareto Frontier、Token Loss、Dial、Waveform。
可选增强 · Recommended
Add-on Loss Curve、Step & Loss、Attribution Score、Inherent Signal。
Add-on Exposure per Vector、Children Layer Scan、Eval Suite。
Add-on White-box / Black-box 模式切换。
Add-on 点击地图点 / token hover / 删除实验 / compare diff 等轻交互。
暂时拒绝 · Defer / Trim
Later 不把所有分析都塞进首屏；避免让 Aleph 看起来像“万能观测台”。
Later 不做太重的后端承诺：比如严格最短、完整 ARCA/GCG、复杂白盒优化。
Later 不让二级面板抢占主要版面；尤其不要取代主滑条或 prompt/output。
Later 不把 context window 当作产品主逻辑，只作为边界参考。
二级面板组件库
这些面板将被拆解为可复用组件，作为 UI/UX 原型里的“secondary panels”。它们必须观测性强、信息密度高、视觉成熟，但又不应该抢主滑条与 Prompt I/O 的核心地位。
01 · Pareto Frontier
Pareto Atlas Panel
用二维地图显式展示离散候选点，而不是让用户被迫相信一条连续曲线。点选后联动右侧 Prompt / Output 面板。
02 · Token Loss
Per-token Loss Heatfield
把 target token / character 与对应 loss 一一映射，形成高级感强的局部热力结构。是白盒模式下最能体现“研究感”的面板之一。
Promptisnotaninstruction.
03 · Loading Dial
Compact Search Dial
用一个小仪表盘表示“正在搜索 / 正在压缩”，视觉角色强，但版面占用极低。适合放在 dashboard 角落或 search controls 中。

04 · Waveform
Cognitive Waveform
借鉴 Aquin / Akyne 的仪器感表达，把输出的内在信号与倾向分布做成“波形 + 频谱”。适合黑盒模式的感性总览。
cognitive_waveform

Optimization Glimpse
不是完整训练面板，而是“搜索过程快照”。用于表达当前 prompt 候选是如何被搜索出来的。
06 · Exposure / Layer / Eval
Safety & Structure Trio

Product premise / 重新收束

Aleph is a compression workbench for model outputs.

核心不是“找到任意 output 的绝对最短 prompt”，而是在固定模型、固定解码、固定度量与固定搜索预算下，估计某个 target output 的最短已知 prompt，并把它从显式复写到最小坐标的压缩路径展示出来。

输入目标 output，拖动压缩路径，看 prompt 如何从“把答案写进去”变成一个可以召唤结果的模型坐标。
Aleph
产品定义Reverse prompt search engine / model-relative compression instrument，不承诺全局最短，只展示 shortest known under budget。
滑条比重每个方案都有滑条，但权重不同：有的是页面骨架，有的是 Dashboard 组件，也有的是仪器控件。
Dashboard 数据Prompt length、Similarity / target fit、Stability、Compression ratio、Leakage score 是基础；部分方案加入 Frontier rank、NLL、latency、cost、judge agreement。
输入 / 输出Target output 永远是起点；Current prompt 与 Model output 永远成对出现，避免只展示 prompt 而看不到模型行为。

How to read these 20 directions

这些不是二十张小卡片，而是二十个可继续展开的单页产品方向。每一页都把 Launching Page、产品说明和实验区压到同一个首屏/主屏里：有目标输入、有压缩路径、有当前 prompt、有模型输出、有 Dashboard，也有字体标注。真实产品可以从其中 2-3 个方向混合：例如用 02 的品牌冲击、01 的工作台结构、04 的 Pareto Map、16 的 deletion ablation 解释层。

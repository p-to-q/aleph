# Project Surfaces

Use this file to prevent accidental overpromising.

| Surface | Status | Owner | Evidence | Notes |
| --- | --- | --- | --- | --- |
| README | Stable | Maintainers | PR review | Public entrypoint. |
| CI | Stable | Maintainers | GitHub Actions | Edit commands per project. |
| ADR/RFC lanes | Optional | Maintainers | Docs review | Enable for strict or research-strict profiles. |
| Agent workflow | Optional but recommended | Maintainers | Handoff review | Enable when agents participate in implementation. |

## Status values

- `stable` — users may rely on it.
- `experimental` — feedback wanted, compatibility not promised.
- `internal` — not a public contract.
- `stub` — visible placeholder; not implemented.
- `archived` — retained for historical receipts.

# Aleph v1.0.1 - 2026-05-19

Aleph v1.0.1 is a small release polish pass on top of the v1 launch surface. It
keeps the same architecture and evidence boundaries, then tightens the public
UI around mode selection, navigation, Markdown rendering, and the compression
axis explanation.

## Highlights

- The default search mode is `fixture`, so first load is deterministic and
  evidence-labeled.
- Typing custom target text automatically switches from `fixture` to
  `custom API`; clearing the input returns to `fixture`.
- The input placeholder now explains all three modes inline:
  `fixture`, `local MLX`, and `custom API`.
- Local MLX guidance is brief and temporary instead of a persistent block of
  text.
- Token-trace unavailable copy is mode-specific: fixture, local MLX, and custom
  API each explain a different evidence boundary.
- The Aleph wordmark opens `https://www.ptoq.io/work`, and the header includes a
  GitHub link to `https://github.com/p-to-q/aleph`.
- Result view has a visible `new search` return control.
- The compression axis uses clearer endpoint labels, while hover/focus tips
  preserve the deeper `K(y|θ)` and explicit-expansion framing.
- The central Markdown renderer now handles nested emphasis such as
  `**William Shakespeare's *Hamlet***` without leaking literal Markdown markers.
- `/health` reports `fixture` alongside hosted or mock search modes, matching
  the actual UI fallback path.

## Evidence Boundaries

- Fixture examples remain precomputed review data.
- Local MLX can expose token text and token NLL only when the adapter returns
  those fields.
- Hosted custom API runs remain black-box observations: prompt and output are
  real, but logits and token NLL are not exposed.
- Out-of-bound slider regions are explanatory spaces, not verified Pareto
  candidates.

apps/api/.venv/bin/python -m pytest apps/api/tests -q
search/.venv/bin/python search/preflight.py

Browser checks on `http://localhost:3002/`:

- Hamlet fixture renders Markdown emphasis as formatted text, not literal `**`
  markers.
- Empty input selects `fixture`.
- Typing custom text selects `custom api`.
- Clearing custom text returns to `fixture`.

## Release Decision

This is suitable as a patch release. It does not change the run contract or
model-adapter architecture. It reduces user confusion around the first-run
workflow and keeps black-box and fixture evidence clearly separated.

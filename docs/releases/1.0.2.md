# Aleph v1.0.2 - 2026-05-19

Aleph v1.0.2 is a patch release on top of v1.0.1. It does not change the run
contract, model-adapter architecture, or evidence boundaries. It adds
production metadata and analytics on the public surface and finishes the
explorer i18n and result-view polish.

## Highlights

- Production metadata: `metadataBase` is `https://aleph.ptoq.io` and the root
  route declares a canonical URL, so shared/indexed links resolve to the
  canonical public site.
- Site icons and link previews: `metadata.icons`, `openGraph`, and `twitter`
  in `web/app/layout.tsx` point at `web/public/aleph-logo.png` (tab favicon and
  share cards; no separate Vercel asset upload).
- Vercel Web Analytics is wired through `web/app/layout.tsx`
  (`@vercel/analytics/next`).
- Explorer i18n and result-view hierarchy are polished: clearer result layout
  and bilingual strings, with the same `AlephRun` surface underneath.
- `README.md` and `DEMO.md` now point at the live site
  (`https://aleph.ptoq.io`).

## Evidence Boundaries

Unchanged from v1.0.1:

- Fixture examples remain precomputed review data.
- Local MLX can expose token text and token NLL only when the adapter returns
  those fields.
- Hosted custom API runs remain black-box observations: prompt and output are
  real, but logits and token NLL are not exposed.
- Out-of-bound slider regions are explanatory spaces, not verified Pareto
  candidates.

## Validation Matrix

Passed locally before tagging:

```bash
npm run lint
npm run test
npm install
npm --workspace web run build
git diff --check
```

Note: the lint/test scripts import `.ts` from `.mjs` via Node ESM, which
requires Node's default type-stripping (Node 22-latest, as pinned by `.nvmrc`
and CI). On Node < 22.18 run them with
`NODE_OPTIONS=--experimental-strip-types`. `web/` build requires `npm install`
first because `@vercel/analytics` is a new dependency in this release.

## Release Decision

Suitable as a patch release. It does not change the run contract or
model-adapter architecture. It improves discoverability (canonical metadata),
adds first-party analytics, and finishes the explorer i18n/result-view polish
without altering fixture, black-box, or local-MLX evidence separation.

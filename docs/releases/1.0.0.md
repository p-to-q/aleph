# Aleph v1.0.0 - 2026-05-19

Aleph v1.0.0 is the first formal release. It is no longer framed as a Hackathon
scaffold or prerelease: the repository now has one active launch surface, a
server-side hosted black-box adapter, real local MLX token traces for the
previously misleading `spring` and `crush` examples, and a release gate that
checks the user-facing app, core contracts, API boundary, and local search
preflight.

The release tag should be `v1.0.0`. Use the date in release notes and receipts,
not as a second competing tag name.

## Highlights

- Active UI path is `web/`, with `apps/web/` retained as archived reference
  material.
- The UI keeps all panels wired through `AlephRun`, `CandidatePoint`, and
  `ObservationSet`.
- `spring` and `crush` now carry real local MLX `toktext`, `toknll`, and
  `targetTokens` fixture traces instead of synthetic distribution-shaped
  fallbacks.
- Missing token traces now render as unavailable evidence instead of fake token
  heatmaps.
- Hosted custom API mode calls an external OpenAI-compatible model through the
  server-side `/api/search` route.
- Hosted runs return real prompt/output candidates while explicitly marking
  observations as `black_box`.
- `/health` reports hosted primary/fallback configuration without exposing
  secrets.
- Result text and token evidence share a bounded scroll area to avoid the
  overlapping left-panel failure seen in screenshots.
- Package versions are aligned to `1.0.0` across the root workspace, `web`, and
  shared packages.
- Next.js was upgraded to `16.2.6`, removing the previous high-severity audit
  finding.

## Hosted API Configuration

The hosted adapter uses server-only environment variables. Keys must be stored
in local untracked env files or deployment environment settings, never in
tracked files.

```bash
ALEPH_CUSTOM_API_BASE_URL=https://zapi.aicc0.com/v1
ALEPH_CUSTOM_API_MODEL=deepseek-v4-pro
ALEPH_CUSTOM_API_KEY=...
ALEPH_CUSTOM_API_FALLBACK_BASE_URL=https://jiuuij.de5.net/v1
ALEPH_CUSTOM_API_FALLBACK_MODEL=qwen3.6-27b-nothinking
ALEPH_CUSTOM_API_FALLBACK_KEY=...
```

The adapter also accepts `ALEPH_OPENAI_BASE_URL`, `ALEPH_OPENAI_MODEL`, and
`ALEPH_OPENAI_API_KEY` for primary-provider compatibility. Public browser
configuration should keep `NEXT_PUBLIC_CLAUDE_API` unset so requests use the
same-origin `/api/search` route.

Production and Development deployment environments have the two server-side
provider configurations set. Preview configuration still needs to be repeated
after the corresponding Git branch exists in the connected deployment project.

## Evidence Boundaries

- Fixture examples are demo data and must stay labeled as fixtures.
- Local MLX token traces support token-level evidence only for runs that include
  token text and token NLL arrays.
- Hosted custom API runs are black-box. They can produce prompts, outputs,
  candidate rankings, retries, and fallback behavior, but not logits or token
  NLL.
- `spring` and `crush` are intentionally honest low-fit real traces. They are
  useful evidence that the UI can show real distributions, not evidence that
  the search has solved those targets well.
- Aleph still claims Shortest Found, not globally shortest possible prompt.

## Known Limits

- Hosted model calls are black-box and can be slow. The local smoke observed
  25.1 second latency for a five-candidate run.
- The hosted route uses conservative repeated sampling for stability and cost;
  deeper reranking remains future work.
- Preview deployment env vars are not configured until the release branch exists
  in the connected deployment project.
- White-box observation contracts are not yet product-stable beyond available
  token NLL traces.
- Non-leaking mode, deletion ablation, prompt-token attribution, saved runs,
  cross-model transfer, and research reports remain future work.
- The large `web/public/aleph-frontier.json` diff is expected because the
  release adds real token traces for `spring` and `crush`.

## Release Decision

This is suitable as a first formal release after the gate passes:

- It has one clear active UI.
- It does not expose API keys.
- It supports public hosted use through server-side credentials.
- It preserves the core Aleph distinctions between Shortest Found, Explicit
  Reconstruction, fixture evidence, black-box evidence, and white-box evidence.
- It records the remaining limits plainly enough that future work can proceed
  from files instead of chat memory.
